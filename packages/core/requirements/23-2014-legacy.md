# R23: 2014 Legacy Content Support

**Status**: 📋 Planned  
**Priority**: P2 (Deprioritized)
**Assignee**: Unassigned  
**Created**: 2026-05-09

---

## Objective

Add support for D&D 5e 2014 legacy content alongside the current 2024 PHB content. This allows users to play with 2014 character options while using the 2024 rules engine.

**Action required**:
- Add 2014 versions of subclasses with `source: "2014 PHB"`
- Mark renamed subclasses with both 2014 and 2024 names
- Ensure `grantedAtLevel` matches 2014 rules

#### 3. Rule Differences Handling

Some rules changed between 2014 and 2024. Need to handle:

| Rule | 2014 | 2024 | Handling |
|------|-------|-------|----------|
| Fighting Styles | Class feature | Feat | Need to support both |
| Species | Race | Species | Already handled via `source` |
| Feat prerequisites | Varies | Standardized | Need validation logic |
| Multiclassing | Different prerequisites | Updated | Need 2014 multiclass rules |
| Spell lists | Some spells moved | Some spells added/removed | Need 2014 spell list |

**Implementation approach**:
- Add `rulesetVersion: '2014' | '2024'` to character options
- Create validation logic that checks `source` compatibility
- Allow mixing 2014 and 2024 content with warnings

## Implementation Plan

### Step 1: Add Legacy Feats (2-3 hours)

1. Create list of 2014 feats with descriptions
2. Add to `static/srd/feats.json` with `source: "2014 PHB"`
3. Write tests for 2014 feat prerequisites
4. Update `src/types/feat.ts` if needed

### Step 2: Add Legacy Subclasses (3-4 hours)

1. Identify all renamed/changed subclasses
2. Add 2014 versions to `static/srd/subclasses.json`
3. Update `src/data/default-loader.ts` to handle both versions
4. Write tests for subclass feature progression

### Step 3: Handle Rule Differences (4-5 hours)

1. Add `rulesetPreference: '2014' | '2024' | 'mixed'` to character creation
2. Update `validateCharacter()` to check source compatibility
3. Add warnings for mixing 2014 and 2024 content
4. Update `levelUp()` to handle 2014 multiclass rules

### Step 4: Testing (2-3 hours)

1. Unit tests for all legacy content
2. Integration tests for mixed 2014/2024 characters
3. Validation tests for rule compatibility

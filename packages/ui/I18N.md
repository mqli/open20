# Internationalization (i18n) Guide

This document explains how to use internationalization in the @open20/ui package and how to extend it in applications like @open20/spellbook.

## Architecture Overview

The i18n system uses a **layered architecture**:

```
@open20/spellbook (Application)
    ↓ extends
@open20/ui (Component Library)
    ↓ provides
Base Translations (common, validation, spellEditor, etc.)
```

- **@open20/ui**: Provides base translations for UI components (~50 keys)
- **@open20/spellbook**: Extends with app-specific translations (spellbook._, characters._, campaigns.\*)
- **Type-safe**: Full TypeScript support with generics
- **Extensible**: Any app can extend base translations with custom namespaces

## Overview

The UI package includes a built-in i18n system that allows you to translate all UI components into different languages. The system is based on React Context and provides hooks for accessing translations.

## Quick Start

### For @open20/ui (Component Library)

```tsx
import { useTranslation } from '@open20/ui';

function MyComponent() {
  const t = useTranslation();

  return <Button>{t('common.cancel')}</Button>;
}
```

### For @open20/spellbook (Application)

```tsx
import { I18nProvider, useTranslation, type BaseTranslations } from '@open20/ui';

// Define extended translations
interface SpellbookTranslations extends BaseTranslations {
  spellbook: { title: string; searchPlaceholder: string };
  characters: { createCharacter: string };
}

// Wrap app with provider
<I18nProvider<SpellbookTranslations> translations={translations}>
  <App />
</I18nProvider>;

// Use in components
const t = useTranslation<SpellbookTranslations>();
t('spellbook.title'); // App-specific
t('common.cancel'); // From UI
```

## Setting Up @open20/spellbook

### Step 1: Define Extended Translations

```typescript
// packages/spellbook/src/i18n/translations.ts
import type { BaseTranslations } from '@open20/ui';

export interface SpellbookTranslations extends BaseTranslations {
  spellbook: {
    title: string;
    searchPlaceholder: string;
    noSpellsFound: string;
    addToSpellbook: string;
  };

  characters: {
    title: string;
    createCharacter: string;
    level: string;
    class: string;
  };
}
```

### Step 2: Create Translation Files

```typescript
// packages/spellbook/src/i18n/locales/en.ts
import type { SpellbookTranslations } from '../translations';

export const en: SpellbookTranslations = {
  // Include all base UI translations
  common: {
    cancel: 'Cancel',
    save: 'Save',
    // ... copy from defaultTranslations
  },

  // Add spellbook-specific
  spellbook: {
    title: 'Spellbook',
    searchPlaceholder: 'Search spells...',
    noSpellsFound: 'No spells found',
    addToSpellbook: 'Add to Spellbook',
  },

  characters: {
    title: 'Characters',
    createCharacter: 'Create Character',
    level: 'Level',
    class: 'Class',
  },
};
```

### Step 3: Create Provider

```typescript
// packages/spellbook/src/i18n/index.ts
import { I18nProvider as BaseI18nProvider } from '@open20/ui';
import { en } from './locales/en';

export function I18nProvider({ children }: { children: React.ReactNode }) {
  return (
    <BaseI18nProvider translations={en}>
      {children}
    </BaseI18nProvider>
  );
}

export { useTranslation } from '@open20/ui';
```

### Step 4: Use in App

```typescript
// packages/spellbook/src/App.tsx
import { I18nProvider } from './i18n';

function App() {
  return (
    <I18nProvider>
      <YourApp />
    </I18nProvider>
  );
}
```

### Step 5: Use in Components

```typescript
// packages/spellbook/src/components/SpellList.tsx
import { useTranslation } from '../i18n';
import { EmptyState, Button } from '@open20/ui';

function SpellList() {
  const t = useTranslation();

  return (
    <div>
      <h1>{t('spellbook.title')}</h1>
      <Button>{t('common.cancel')}</Button>
      <EmptyState title={t('spellbook.noSpellsFound')} />
    </div>
  );
}
```

## Adding New Languages

### Providing Custom Translations

You can provide your own translations by passing them to the I18nProvider:

```tsx
import { I18nProvider, defaultTranslations } from '@open20/ui';

const spanishTranslations = {
  ...defaultTranslations,
  common: {
    ...defaultTranslations.common,
    cancel: 'Cancelar',
    save: 'Guardar',
    saving: 'Guardando...',
    // ... other translations
  },
  // ... other translation sections
};

function App() {
  return (
    <I18nProvider translations={spanishTranslations}>
      <YourComponents />
    </I18nProvider>
  );
}
```

## Available Hooks

### useTranslation

Returns the translation function `t`:

```tsx
const t = useTranslation();
const text = t('common.cancel'); // "Cancel"
```

### useI18n

Returns the full i18n context:

```tsx
const { t, locale, setLocale, translations } = useI18n();
```

## Adding New Translations

To add support for a new language:

1. Create a new translation object based on `defaultTranslations`
2. Translate all strings to your target language
3. Pass the translations to `I18nProvider`

Example for French:

```tsx
const frenchTranslations = {
  common: {
```

## Best Practices

1. **Always use translation keys** instead of hardcoded strings in components
2. **Provide fallback values** when using translations
3. **Test with different languages** to ensure proper layout and text length
4. **Keep translation keys consistent** across the application
5. **Use descriptive key names** that indicate the context

## Migration Guide

If you have existing components with hardcoded strings:

1. Import `useTranslation` hook
2. Replace hardcoded strings with `t('key.path')` calls
3. Add the translation keys to your translation files
4. Test the component with different languages

Example migration:

```tsx
// Before
<Button>Cancel</Button>;

// After
const t = useTranslation();
<Button>{t('common.cancel')}</Button>;
```

## Troubleshooting

### Missing Translation Keys

If a translation key is not found, the system will:

1. Log a warning to the console
2. Return the key itself as fallback

### Hook Usage Errors

Make sure to use i18n hooks within an `I18nProvider`:

```tsx
// ❌ Wrong - hook used outside provider
function Component() {
  const t = useTranslation(); // Error!
  return <div>{t('common.cancel')}</div>;
}

// ✅ Correct - hook used within provider
function App() {
  return (
    <I18nProvider>
      <Component />
    </I18nProvider>
  );
}

function Component() {
  const t = useTranslation(); // Works!
  return <div>{t('common.cancel')}</div>;
}
```

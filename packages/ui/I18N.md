# Internationalization (i18n) Guide

This document explains how to use internationalization in the `@open20/ui` package and how to extend it in applications like `@open20/spellbook`.

## Architecture Overview

The i18n system uses a layered architecture:

```txt
@open20/spellbook (Application)
    extends
@open20/ui (Component Library)
    provides base translations
```

- `@open20/ui` provides base translations for shared UI components
- Apps like `@open20/spellbook` provide app-specific keys on top
- TypeScript generics provide typed access to extended translation shapes

## Overview

The UI package includes a built-in i18n system based on React Context.

- `I18nProvider` provides locale state + translation set
- `useTranslation()` returns a translation function `t`
- `useI18n()` returns full context (`t`, `locale`, `setLocale`, `translations`)

## Quick Start

### In `@open20/ui` components

```tsx
import { useTranslation } from '@open20/ui';

function MyComponent() {
  const t = useTranslation();
  return <button>{t('common.cancel')}</button>;
}
```

### In `@open20/spellbook`

```tsx
import { I18nProvider } from '@open20/ui';
import { enTranslations, zhCNTranslations } from './i18n';

<I18nProvider
  initialLocale="en"
  translationsSet={{
    en: enTranslations,
    'zh-CN': zhCNTranslations,
  }}
>
  <App />
</I18nProvider>;
```

## Setting Up `@open20/spellbook`

### Step 1: Define extended translations

```ts
// packages/spellbook/src/i18n/index.tsx
import type { BaseTranslations } from '@open20/ui';

export interface SpellbookTranslations extends BaseTranslations {
  spells: string;
  searchSpells: string;
  createCharacter: string;
  noSpellsFound: string;
  cast: string;
  // ...other spellbook keys
}
```

### Step 2: Create locale objects

```ts
// packages/spellbook/src/i18n/index.tsx
import { defaultTranslations } from '@open20/ui';

export const enTranslations = {
  ...defaultTranslations,
  spells: 'Spells',
  searchSpells: 'Search spells...',
  createCharacter: 'Create Your Hero',
  noSpellsFound: 'No spells found matching your criteria.',
  cast: 'Cast',
};
```

### Step 3: Use provider in app

```tsx
// packages/spellbook/src/App.tsx
import { I18nProvider } from '@open20/ui';
import { enTranslations, zhCNTranslations } from './i18n';

export function App() {
  return (
    <I18nProvider
      initialLocale="en"
      translationsSet={{
        en: enTranslations,
        'zh-CN': zhCNTranslations,
      }}
    >
      <YourApp />
    </I18nProvider>
  );
}
```

### Step 4: Use in components

```tsx
import { useTranslation } from '@open20/ui';

function SpellList() {
  const t = useTranslation();

  return (
    <div>
      <h1>{t('spells')}</h1>
      <button>{t('common.cancel')}</button>
      <p>{t('noSpellsFound')}</p>
    </div>
  );
}
```

## Adding New Languages

`I18nProvider` currently supports `en` and `zh-CN` locale keys through `translationsSet`.

```tsx
import { I18nProvider, defaultTranslations, zhCNTranslations } from '@open20/ui';

const customEn = {
  ...defaultTranslations,
  common: {
    ...defaultTranslations.common,
    cancel: 'Abort',
  },
};

function App() {
  return (
    <I18nProvider
      initialLocale="en"
      translationsSet={{
        en: customEn,
        'zh-CN': zhCNTranslations,
      }}
    >
      <YourComponents />
    </I18nProvider>
  );
}
```

## Available Hooks

### `useTranslation`

```tsx
const t = useTranslation();
const text = t('common.cancel');
```

### `useI18n`

```tsx
const { t, locale, setLocale, translations } = useI18n();
```

## Components Currently Using i18n

- `SpellEditor`: labels, validation messages, buttons
- `EmptyState`: fallback title/description text
- `ThemeToggle`: tooltip text
- `Dialog.Close`: close button text
- `SpellCard`: level/meta/action labels

## Best Practices

1. Use translation keys instead of hardcoded UI strings
2. Keep key naming consistent across app + shared UI
3. Test layouts in both `en` and `zh-CN`
4. Prefer extending `BaseTranslations` in app-level interfaces

## Troubleshooting

### Missing translation key

If a key is missing:

1. A warning is logged in the console
2. The key string is returned as fallback

### Hook used outside provider

`useTranslation()` and `useI18n()` must run under `I18nProvider`.
